#### 腾讯云用户

```
登录: 使用腾讯云账户
导航: 云服务器 → 实例列表

```

#### AWS用户
```
URL: https://console.aws.amazon.com/
登录: 使用AWS账户
导航: EC2 → Instances
搜索: 113.45.64.145
```

### 第2步: 配置安全组规则 (3-5分钟)

#### 阿里云配置
1. 点击实例ID进入详情页
2. 点击"安全组"标签
3. 点击"配置规则"
4. 添加入站规则:
   ```
   规则方向: 入站
   授权策略: 允许
   协议类型: 自定义TCP
   端口范围: 8000/8000
   授权对象: 0.0.0.0/0
   优先级: 1
   描述: AI Resume Backend API
   ```
5. 点击"确定"

#### 腾讯云配置
1. 点击实例ID
2. 点击"安全组"→"修改规则"
3. 添加入站规则:
   ```
   类型: 自定义
   来源: 0.0.0.0/0
   协议端口: TCP:8000
   策略: 允许
   备注: AI Resume Backend API
   ```
4. 保存规则

#### AWS配置
1. 选择实例→Actions→Networking→Change security groups
2. 或编辑Security Group入站规则:
   ```
   Type: Custom TCP
   Port Range: 8000
   Source: 0.0.0.0/0
   Description: AI Resume Backend API
   ```
3. Save rules

### 第3步: 等待规则生效 (1-2分钟)

⏰ **重要**: 安全组规则通常需要1-2分钟生效

在此期间:

## ❌ 故障排查

### 问题1: 规则配置后仍无法访问

**可能原因**:
- 规则未完全生效
- 多层防火墙配置
- 网络ACL限制

**解决方案**:
```bash
# 1. 等待更长时间 (最多5分钟)
sleep 300

# 2. 检查是否有其他安全组
# 在控制台查看所有关联的安全组

# 3. 检查网络ACL
# 在VPC控制台查看网络ACL规则

# 4. 联系云服务商技术支持
# 提供实例IP和配置的规则
```

### 问题2: 找不到安全组配置

**可能原因**:
- 权限不足
- 实例不在预期账户下
- 使用了其他云服务商

**解决方案**:
```bash
# 1. 检查实例详情
# 确认实例归属账户

# 2. 联系服务器管理员
# 确认有权限操作该实例

# 3. 尝试其他云服务商控制台
# AWS、阿里云、腾讯云等
```

### 问题3: 配置后出现其他问题

**紧急回滚**:
```
1. 在安全组中删除刚添加的规则
2. 恢复到配置前状态
3. 检查容器日志: docker logs ai-resume-backend
4. 重启容器: docker restart ai-resume-backend
```
