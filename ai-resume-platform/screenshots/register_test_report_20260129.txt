========================================
四、发现的问题
========================================

【严重问题】Backend Bug #1: bcrypt版本不兼容
----------------------------------------
错误信息: password cannot be longer than 72 bytes
原因: bcrypt 5.0.0 与 passlib 1.7.4 不兼容
影响: 导致注册功能完全无法使用
状态: ✓ 已修复
解决方案: 降级bcrypt到3.2.2版本
命令: pip install 'bcrypt<4.0.0'

【严重问题】Backend Bug #2: 数据库autoincrement不工作
----------------------------------------
错误信息: NOT NULL constraint failed: users.id
原因: SQLite对BigInteger类型的autoincrement支持有限
影响: 用户无法注册，数据库插入失败
状态: ✓ 已修复
解决方案: 将User.id列从BigInteger改为Integer类型
文件: app/models/user.py

【严重问题】Backend Bug #3: Favorite表缺少外键约束
----------------------------------------
错误信息: Could not determine join condition
原因: Favorite表的user_id列没有定义ForeignKey
影响: ORM关系映射失败，无法操作数据库
状态: ✓ 已修复
解决方案: 
1. 添加导入: from sqlalchemy import ForeignKey
2. 修改列定义: user_id = Column(BigInteger, ForeignKey("users.id"), ...)
3. 修改关系: user = relationship("User", back_populates="favorites")
文件: app/models/template.py

【中等问题】前端 TODO项
----------------------------------------
问题: 用户协议和隐私政策链接未实现
影响: 用户无法查看协议内容
建议: 添加协议页面或弹窗显示协议内容
优先级: 中

----------------------------------------

修复 #2: User.id列类型修改
----------------------------------------
原定义: id = Column(BigInteger, primary_key=True, autoincrement=True)
新定义: id = Column(Integer, primary_key=True, autoincrement=True)
影响: 所有使用BigInteger作为ID的模型都需要修改
建议检查文件:
- app/models/resume.py
- app/models/template.py
- app/models/operation_log.py

修复 #3: Favorite外键约束
----------------------------------------
修改前:

修改后:
user_id = Column(BigInteger, ForeignKey("users.id"), nullable=False, index=True)
